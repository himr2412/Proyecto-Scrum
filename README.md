# Proyecto-Scrum
Proyecto que consta de dos esprints que tiene con finalidad la interfaz de la aplicación y el skillmaching

## Objetivos del Proyecto
> [!NOTE]
>Este proyecto esta esta limitado con el tiempo y el presupuesto asi que solo constara por ahora de dos Sprints.

Resumen breve de nuestro documento sobre la `Aplicacion estilo Red Social sobre juegos de Mesa`.
```
1.Definicion del Problema.
2.Que quiere el Cliente.
3.Historas de Usuario.
4.
5.
```
### Características 

>Al ser uno de las `primeras metodologías` en originarse y ser la base de todos los demás modelos de ciclo de vida:

  - Línea y secuencia. 
  - Promueve una metodología de trabajo efectiva.
  - Basada en documentación.
> [!IMPORTANT]
>Sin embargo, también presentan sus `desventajas`:

  - Rigidez
  - Fases distintas

> [!CAUTION]
> La metodología en cascada es **rígida**: una vez cerrada una fase,  
> es difícil volver atrás sin afectar tiempos y presupuesto.

### Fases principales
1. **Análisis**
   - Recopilar requisitos funcionales y no funcionales.
   - Identificar actores y casos de uso.
   - Documentar especificaciones del sistema.
   - Resultado: modelo claro de lo que debe hacer el software.
> [!WARNING]
> Si los requisitos no se definen correctamente en la fase de **análisis**,  
> el proyecto puede fallar en fases posteriores y generar **costes elevados**.

2. **Diseño**
   - Definir arquitectura, diagramas y estructura de datos.
3. **Implementación**
   - Programar el sistema según el diseño.
4. **Pruebas**
   - Verificar que el software cumple los requisitos.
5. **Mantenimiento**
   - Corregir errores y realizar mejoras tras la entrega.

---


> [!TIP]
> Antes de comenzar cada fase, asegúrate de que la anterior esté **bien documentada**.  
> Esto evita retrabajos y facilita que el equipo entienda claramente los objetivos.
